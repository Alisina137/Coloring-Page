Replace the entire image generation system in this project. Remove all existing image generation providers and their integrations, including:

* Gemini API
* OpenAI Images API
* Any existing Hugging Face implementation (replace it with the new standardized implementation below)
* Any unused image generation code, services, API clients, environment variables, and dependencies

Implement a new provider-based image generation architecture with the following providers, in this priority order:

1. Hugging Face (Primary)
2. Cloudflare Workers AI (Fallback #1)
3. Pollinations AI (Fallback #2)
4. fal.ai (Fallback #3)

## Requirements

### Provider Architecture

Create a clean provider abstraction.

Example:

* ImageGenerator interface
* HuggingFaceProvider
* CloudflareProvider
* PollinationsProvider
* FalProvider
* ImageGenerationService

The application should never directly call provider APIs. All requests should go through ImageGenerationService.

---

### Failover Logic

When generating an image:

1. Try Hugging Face first.
2. If it fails (timeout, rate limit, quota exceeded, HTTP error, invalid response), automatically retry with Cloudflare Workers AI.
3. If Cloudflare fails, automatically retry with Pollinations.
4. If Pollinations fails, automatically retry with fal.ai.
5. Only show an error if all four providers fail.

This failover must happen automatically without user interaction.

---

### Prompt

Use a prompt optimized for coloring books.

Base prompt:

"A cute {subject}, coloring book page, black outline only, clean thick line art, no shading, no grayscale, no color, white background, closed shapes, printable, vector-style illustration, suitable for children, centered composition, high contrast."

Append negative instructions whenever supported:

"No shadows, no gradients, no textures, no cross hatching, no background clutter, no realistic lighting, no fill colors."

### fal.ai

### Timeouts

Set a timeout of 60 seconds per provider.

If a provider exceeds the timeout, move to the next provider automatically.

---

### Retry Logic

Retry once for transient errors before moving to the next provider.

Do not retry on invalid API keys.
